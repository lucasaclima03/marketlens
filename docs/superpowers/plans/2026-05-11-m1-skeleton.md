# M1 — Repo Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap the MarketLens repository with a sound, testable, observable NestJS skeleton — two process modes (API + Worker), Postgres + Redis connectivity, `/health` endpoint, ESLint Tier S quality gates, Vitest with coverage thresholds, multi-stage Dockerfile, GitHub Actions CI, pre-commit hooks. **Zero domain logic.** M1 is done when the application boots, the health endpoint responds, all CI jobs pass, and a developer can clone-install-run in under 10 minutes.

**Architecture:** Single TypeScript codebase, two Node entrypoints (`main-api.ts`, `main-worker.ts`) sharing the same `src/`. NestJS 11 modules organized by feature (M1 only has shared cross-cutting modules: `config`, `db`, `logging`, `health`, `bullmq`). Drizzle ORM with `postgres-js` driver; BullMQ 5 over Redis; `nestjs-pino` for structured logs. Multi-stage Dockerfile producing a single image that runs both processes via `CMD` override.

**Tech Stack:** Node 22 LTS, TypeScript 5.6, NestJS 11, Drizzle ORM, BullMQ 5, Postgres 16, Redis 7, Vitest 2, ESLint 9 (flat config) + Prettier 3, Husky + lint-staged + commitlint, GitHub Actions, Docker + docker-compose.

**Reference spec:** `docs/superpowers/specs/2026-05-11-skeleton-and-first-vertical-slice-design.md` — when this plan and the spec disagree, the spec wins.
